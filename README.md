# SimpleNote

Laravel 学習用に作成した **シンプルなメモアプリ** です。  
CRUD の基本、認証（Auth）、Controller → View のデータ受け渡し、  
タグによる分類機能を一通り実装しています。

本アプリは「機能を増やすこと」よりも、  
**Laravel の基本構造と設計の理解**を目的として作成しました。

---

## 🎯 目的

* Laravel の基本構造（MVC）を理解する
* Controller の責務分離を意識した設計を行う
* 面接で「なぜこう作ったか」を説明できる成果物を作る

※ 講座（Laravel 6 前提）を参考にしつつ、  
※ **Laravel 12 環境向けに構成・実装を見直しています**

---

## 🛠 使用技術

* PHP 8.3
* Laravel 12
* MySQL
* Bootstrap
* Docker（Laravel Sail）

※ 学習効率と実務での使用頻度を考慮し、  
※ Laravel 標準構成を中心に技術選定しています。

---

## ✨ 主な機能

### 認証機能

* ユーザー登録
* ログイン / ログアウト
* 未ログインユーザーは `/home` にアクセス不可

### メモ機能

* メモ作成
* メモ一覧表示
* メモ詳細表示
* メモ編集
* メモ削除

### タグ機能

* メモ作成・編集時にタグを入力可能
* 同名タグは自動的に再利用（`firstOrCreate`）
* タグ一覧表示
* タグクリックによるメモ絞り込み

※ タグは **メモを探しやすくするための補助情報** として扱っています。

---

## 🧩 タグ設計について

### 設計方針

* タグはメモの「補助情報」であり、主役ではない
* タグ単体の CRUD 画面は持たせない
* UI や責務を増やしすぎないことを優先

### 採用した構成

* メモとタグは **1対多** の関係
* 今回は学習目的のため、多対多構成は採用していません

### 仕様

* タグはメモ作成・編集時にのみ入力
* タグ名の編集画面は設けない
* タグ削除時は、紐づくメモの `tag_id` を `null` に設定

---

## 🏗 画面構成

* **welcome**
  * ログイン・新規登録への導線のみを提供

* **home**
  * 左：タグ一覧
  * 中央：メモ一覧
  * 右：新規メモ作成フォーム

HomeController は「表示専用」とし、  
CRUD 処理および業務ロジックは MemoController に集約しています。

---

## 🗄 データベース構成（概要）

* **users**
  * ユーザー情報

* **memos**
  * メモ本体
  * `user_id`, `tag_id` を保持

* **tags**
  * タグ情報
  * ユーザー単位で管理

---

## 🧠 設計・実装で意識した点

* Controller の責務を明確に分離
  * HomeController：一覧・表示専用
  * MemoController：CRUD および業務ロジック
* resource ルートに含まれない処理（タグ絞り込み等）は用途別メソッドとして定義
* `Auth::id()` を用いたユーザー単位のデータ分離
* 他ユーザーのデータにアクセスできないよう所有者チェックを実装
* View には表示ロジックのみを記述し、処理は Controller に集約

---

## 🔐 セキュリティ・権限制御

* 全画面に認証ミドルウェアを適用
* メモ・タグ操作時に必ずログインユーザーとの紐付きを確認
* 不正アクセス時は `403 Forbidden` を返却
